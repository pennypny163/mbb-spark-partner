# MBB Spark Partner ⚡

> 不是给你答案的 AI，是跟你一起想出答案的合伙人。

一个 WorkBuddy Skill，把 AI 变成 MBB（麦肯锡/贝恩/波士顿咨询）级别的战略思考搭档。它不会安静地写一份 50 页报告然后发给你——它会拉着你一起想，一边推理一边追问，直到你们共同到达一个清晰的决策。

---

## 它不是什么

- ❌ 不是"帮你写报告"的工具（那个用 [mbb-consultant-distiller](https://github.com/pennypny163/mbb-consultant-distiller)）
- ❌ 不是给一堆通用建议让你自己消化的 ChatBot
- ❌ 不是问你 10 个问题才开始思考的问卷系统

## 它是什么

- ✅ 一个会**秀出推理过程**的思考伙伴——你能看到它怎么想的
- ✅ 一个**给选择题不给开放题**的追问者——永远是 A/B/C，不是"你觉得呢？"
- ✅ 一个**敢追问你内部信息**的合伙人——"你们留存率大概多少？给个区间就行"
- ✅ 一个**敢说'别做了'**的诤友——不会因为你兴奋就陪你high

---

## 一次完整的对话长什么样

```
你：我想做一个给泡泡玛特盲盒机的供应链优化 agent，这事值得做吗？

Spark Partner：
  → 自动识别这是一个 Market Entry 问题
  → "你跟泡泡玛特的关系目前是 A)有关系人 B)纯外部 C)体系内？"

你：A

Spark Partner：
  → 分析维度1（需求真实性），暴露推理链
  → 唱一个反调
  → "他们最痛的是 A)热门断货 B)冷门压货 C)补货人力跟不上？"

你：C

Spark Partner：
  → "有意思。那你的 agent 真正卖的不是'预测'，是'省人'。"
  → 调整分析框架，继续推进...
  
  ...3-5轮后...

  → 给出旗帜鲜明的结论 + 周一就能开始做的行动建议
  → Recap：回顾方法论 + 点出你的决策风格/盲区 + 一句个性化收尾
```

## 核心设计原则

### 1. 思考透明 > 结论华丽

每一步分析都暴露推理链：

```
"我沿着两条线想了一下：
 - 如果核心诉求是速度 → 方案A，3个月落地，但整合风险高
 - 如果核心诉求是控制力 → 方案B，12个月，但长期可控
 
 考虑到你说资金紧 → 我倾向B。你觉得呢？"
```

### 2. 给选项，不给开放题

```
❌ "你觉得你们的核心竞争力是什么？"
✅ "你们的护城河可能是：A)技术壁垒 B)渠道壁垒 C)品牌心智 — 哪个最接近？"
```

### 3. 敢追问内部信息

网上搜不到的东西直接问：

- 团队核心能力分布
- 现金流够烧多久
- 老板的真实偏好和禁区
- 竞争对手的未公开动作

### 4. 红灯就劝退

三个维度分析完，两个亮红灯→直接说"不建议做"：

```
"坦白说，我的判断是：这个现在不该做。
 - 需求优先级不够高
 - 切入壁垒比想象的高
 唯一能翻转的条件是 [X]。你有这个条件吗？"
```

### 5. 每条建议通过"周一测试"

不说"进一步研究市场"这种废话。说：

```
"本周跟关系人约30分钟电话，核心目标：确认他们内部有没有在建的类似项目。"
```

## 完整流程

```
Phase 1 → 定义问题 + 自动匹配框架
           "这本质是个 Market Entry 问题，我用这个框架来拆——"

Phase 2 → 逐维度分析 + 持续碰撞
           每个维度：判断 + 推理链 + 唱反调 + 追问确认
           每2个维度做一次"线索拼图"小结

Phase 3 → 收敛结论 + 行动建议
           旗帜鲜明的决策 + 可执行的 next steps + 风险标注

Phase 4 → Recap
           方法论回顾 + 你的决策风格/盲区 + 一句个性化 ending
```

**流程不是铁轨**——你随时可以说"直接给结论"、"跳过这个"、"我不同意"，它会立即适应。

## 覆盖的分析框架

自动匹配，不需要你选：

| 你的问题类型 | 自动用的框架 |
|-------------|------------|
| "要不要做/要不要进" | Market Entry + Risk Assessment |
| "怎么增长/怎么打" | GTM Strategy + Pricing |
| "竞品怎么样" | SWOT + Porter's Five Forces |
| "怎么定价/变现" | Pricing + Unit Economics |
| "风险有多大" | Risk & Scenario Planning |
| "卖给谁" | Personas + Customer Journey |
| "市场多大" | TAM Sizing (Top-down + Bottom-up) |
| "什么趋势/风口" | Industry Trends + Market Sizing |
| "怎么转型" | Executive Synthesis (全面诊断) |

## Recap 长什么样

分析结束后，自动附上：

```
📋 Recap

方法论：我们用 Market Entry 框架，沿着需求→切入→复制三条线推进。

关于你：
- 你做选择很快——说明内心已有倾向，只是在找外部确认
- 你全程没问过 downside——建议花5分钟想想最坏情况
- 你关注运营可行性而非市场叙事——这个视角很务实

去吧。最坏的结果不是失败，是三个月后你知道了一个别人不知道的答案。
```

## 安装

本 Skill 是一个纯 Markdown 指令集，不依赖任何特定平台。只要你的 AI 工具支持加载自定义 prompt/skill 文件，就能用。

### 方式一：Claude Code / Cline / Cursor 等支持 skills 的工具

```bash
# 克隆到你的 skills 目录（路径根据你的工具调整）
git clone https://github.com/pennypny163/mbb-spark-partner.git ~/.your-tool/skills/mbb-spark-partner
```

### 方式二：直接喂给任何 AI

1. 复制 `SKILL.md` 的全部内容
2. 作为 system prompt 或对话开头粘贴给 ChatGPT / Claude / 任何 LLM
3. 开始对话即可

### 方式三：WorkBuddy 用户

```bash
# 用户级别（所有项目可用）
cp -r mbb-spark-partner ~/.workbuddy/skills/

# 或项目级别（团队共享）
cp -r mbb-spark-partner ./.workbuddy/skills/
```

**核心文件就一个**：`SKILL.md`（461行）。其余都是辅助。
